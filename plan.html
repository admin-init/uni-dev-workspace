1. Overview & Design Principles

uni-dev is a universal backend development framework that automates the full software development lifecycle through multi-agent orchestration. It is not tied to any single project, language, or framework—it is a methodology encoded into an agent system.

① Specification First

OpenAPI contract written before any implementation code. The spec is the source of truth. Code must conform.

② Test Before Code

Contract tests + unit tests written before implementation. Code is judged by whether tests pass.

③ Domain Model First

Entities, aggregates, value objects, bounded contexts identified before technical design.

④ Deterministic Control

Verification gates, retry limits, iteration ordering — all handled by deterministic Python, never LLM.

⑤ LLM for Intelligence

Code generation, architectural decisions, specification writing — handled by LLM sub-agents.

⑥ Single Source of Truth

The Knowledge Base is the canonical representation of the codebase. Agents query it; they never trust their own memory.

3. Deterministic LangGraph Core

Critical Rule These nodes are pure Python. Zero LLM calls. They cannot be overridden, hallucinated, or skipped. ~50 lines total per node.

# Node Logic LOC
① Verification Gate Reads MCP compare_api_responses() + verify_contract() + test exit code.
Returns pass or fail. LLM cannot override test results.		 ~16
② Retry Controller Counter: attempts < 3 ? "retry" : "escalate_to_human".
LLM cannot argue for a 4th try.		 ~10
③ Migration Stepper Array index: idx += 1; return "done" if idx >= len(controllers) else "next".
LLM cannot skip or reorder.		 ~11
④ Classification Router Dict lookup: routes.get(classification, default).
LLM classifies; code routes. Routing table is immutable.		 ~13

Workflow StateGraph

[START] ──► ClassificationRouter ──► "add_feature" → DomainDesigner → SpecWriter → TestGenerator → CodeGenerator → VerificationGate ──► "update_api" → SpecWriter → TestGenerator → CodeGenerator → VerificationGate ──► "remove" → DomainDesigner → SpecWriter → CodeGenerator → VerificationGate VerificationGate ──► "pass" → Reviewer → [END] ──► "fail" → RetryController ──► "retry" → CodeGenerator (back to implementation) ──► "escalate_to_human" → [END]

State Schema

class UniDevState(TypedDict):
    messages:         list[BaseMessage]

4. Knowledge Base Design

Scope: Project-specific. Each repository gets its own .uni-dev/ directory containing SQLite, ChromaDB, and graph data.

tree-sitter Parsers

Source code parsed into structured JSON via plugin-based language parsers. Shipped: Java + Node.js.

SQLite (8 tables)

Relational store for modules, classes, methods, endpoints, DB tables/columns, UI components.

ChromaDB (14 indexes)

Embedding + BM25 keyword indexes for semantic and exact search across code, specs, and docs.

NetworkX Graph

8 node types, 11+3 edge types. Dependency graph for migration planning.

5. Parser Extension System

Plugin Architecture Each language parser is a standalone plugin implementing the ParserPlugin interface. Plugins are discovered via Python entry points and loaded on demand.

ParserPlugin Interface

class ParserPlugin(ABC):
    """Base class for all language parsers."""

    @abstractmethod
    def language(self) → str:
        """Return language identifier: 'java', 'typescript', etc."""
        ...

    @abstractmethod
    def detect(self, file_path: str) → bool:
        """Return True if this parser should handle the file."""
        ...

    @abstractmethod
    def parse(self, file_path: str, source: str) → ParseResult:
        """Parse source code into structured JSON."""
        ...

Shipped Parsers

Java / Spring Boot

unicore-parser-java

  • ControllerParser — extracts @RestController, @GetMapping, etc.
  • ServiceParser — extracts @Service, method signatures
  • EntityParser — extracts @Entity, @Column, JPA annotations
  • MapperParser — extracts MyBatis XML mapper queries

Node.js / Express / NestJS

unicore-parser-nodejs

  • RouteParser — extracts Express router.get(), NestJS @Get()
  • ServiceParser — extracts service classes and methods
  • ModelParser — extracts Sequelize/TypeORM model definitions
  • MiddlewareParser — extracts auth middleware chains

7. LLM Sub-Agents

domain-designer

DDD Phase

Identifies entities, aggregates, value objects, and bounded contexts from the issue description and existing codebase.

Tools: search_code, get_class_structure, get_dependency_graph

Output: Domain model YAML

spec-writer

SDD Phase

Writes or updates OpenAPI 3.0 YAML contracts based on the domain model. The spec is the source of truth.

Tools: get_api_contract, get_entity_spec, filesystem tools

Output: OpenAPI YAML

code-generator

TDD Phase

Implements code conforming to the spec. Uses test failures to guide corrections.

Tools: filesystem tools, shell (for builds/tests), get_business_logic_doc

Output: Source files

test-generator

TDD Phase

Generates contract tests and unit tests from specs. Tests are written before implementation.

Tools: get_api_contract, verify_contract, shell

Output: Test files

reviewer

Post-Verification

Reviews implementation against spec, generates documentation updates, identifies gaps.

Tools: All MCP tools, compare_api_responses, get_migration_checklist

Output: Review report + doc updates

8. Security Filter

Critical Middleware that intercepts tool call arguments and results to redact sensitive information before it reaches logs or the knowledge base. 

class LogFilter:
    """Redact PII, secrets, and credentials from agent I/O."""

    PATTERNS: list[tuple[str, str]] = [
        (r'Bearer\s+[A-Za-z0-9\-._~+/=]+',    'Bearer <REDACTED>'),
        (r'api[_-]?key[=:]\s*[^\s&]+',         'api_key=<REDACTED>'),
        (r'password[=:]\s*[^\s&]+',            'password=<REDACTED>'),
        (r'[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}', '<EMAIL>'),
        (r'postgresql://[^@]+@',                'postgresql://<CREDENTIALS>@'),
        (r'mongodb://[^@]+@',                    'mongodb://<CREDENTIALS>@'),
        (r'REDIS_URL[=:]\s*[^\s]+',             'REDIS_URL=<REDACTED>'),
    ]

    def filter_text(self, text: str) → str:
        for pattern, replacement in self.PATTERNS:
            text = re.sub(pattern, replacement, text, flags=re.IGNORECASE)
        return text

    # Plugs into deepagent middleware as a tool-call wrapper
    # All tool args and results pass through filter before logging/KB insertion
