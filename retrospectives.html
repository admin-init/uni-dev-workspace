Key Takeaways for Future Milestones

  • Regex-based parsing is fragile. Nested parens, cross-method matching, and class vs. method ambiguity caused most bugs. Consider tree-sitter AST for complex parsing.
  • Test fixtures must be realistic. SQL strings with (), annotations with hasRole('ADMIN') — edge cases only surfaced through thoughtful test data.
  • Registry auto-load is a side effect. Query methods should not trigger discovery — find() is the only operation that needs full scan.
  • Token scope matters. HTTPS pushes of workflow files require workflow OAuth scope. SSH bypasses this entirely.
  • uvx for lint. uv run --with <tool> installs the whole project; uvx runs the tool in isolation.
  • Conditional deps are real. importlib_metadata is stdlib on Python ≥ 3.12, but 3.11 needs the backport explicitly.

Key Takeaways for Future Milestones

  • ChromaDB tests need real embedding engine. DefaultEmbeddingFunction downloads all-MiniLM-L6-v2 on first use (~80MB). Tests are correct but slow on first run. Consider a fixture that pre-initializes.
  • sqlite-utils auto-creates tables. The initial create_tables() call with pk= sets up the schema; subsequent opens reuse it. No separate migration step needed.
  • Graph edges are direction-aware. nx.DiGraph() edges point from source to target. HAS_METHOD goes Class→Method; ROUTES_TO goes Class→Endpoint; FK_TO goes Column→Table. Consistent naming is critical for topological_sort() to be meaningful.
  • CLI walks are I/O-bound. Path.rglob() across large projects will be slow. Consider parallel parsing in M4.
  • ChromaDB persistence path is absolute. Using tempfile.TemporaryDirectory in tests works but edge cases around ChromaDB background threads require careful teardown.

Key Takeaways for Future Milestones

  • JavaScript/TypeScript parsing is fundamentally different from Java. No annotations in the JVM sense — decorators, middleware chains, and module.exports require entirely different regex patterns.
  • TypeScript return types break method regex. : Promise<User> after params requires optional type-annotation skipping in the method signature pattern.
  • Empty-source guard is a universal pattern. Every parser should return empty ParseResult() on blank source. This should be codified in the ParserPlugin base class contract.
  • Sequelize and TypeORM diverge significantly. Field definitions, table naming, and relationship modeling use completely different syntax. Both need dedicated extraction paths with shared output format.

Key Takeaways for Future Milestones

  • Pattern ordering matters for regex redaction. Specific patterns (DB URLs) must run before generic patterns (email) to prevent partial matches. The first match wins.
  • Replacement strings don't preserve case. re.IGNORECASE only affects matching, not substitution. The replacement is always the literal string.

  • filter_dict() is recursive and type-safe. Handles nested dicts, lists, and mixed structures. Non-string values pass through unchanged.
  • 20 tests cover all 7 patterns. Each pattern tested individually, plus multi-pattern interaction, edge cases, empty inputs, and false positive avoidance.

Key Takeaways for Future Milestones

  • wrap_tool_call is the ideal interception point. It fires for every tool invocation — perfect for logging, monitoring, and the security LogFilter. The pattern is consistent across deepagents.
  • State reducers are essential for append-only fields. Without a custom reducer, state updates replace rather than accumulate. The Annotated[NotRequired[T], reducer] pattern is well-documented in LangGraph.
  • MonitorMiddleware runs outermost. By prepending to the middleware list, it wraps all other middleware — seeing every tool call before filesystem, sub-agent, or skills middleware processes it.
  • SQLite with auto-schema is lightweight and reliable. CREATE TABLE IF NOT EXISTS ensures zero-migration startup. sqlite3.Row factory gives clean dict results.
  • Three output channels serve different consumers. State for in-session querying, SQLite for historical analysis, logging+streams for real-time dashboards. Each has different latency/queryability tradeoffs.

# Architecture change
# Before: graph.py called LLMs directly → no tools, fabricated test_results
# After:  graph.py produces instructions → LLM drives loop via task() → full tools
# New nodes: EscalationRouter (anti-loop blocked routing) + VerificationGate (blocked state)

Key Takeaways for Future Milestones

  • Two-runtime architecture is a design flaw. CompiledSubAgent in deepagents cannot call parent's sub-agents. Don't nest LangGraph runtimes expecting upward delegation.
  • Instruction-producing state machines work. Graph as pure state→action converter, driven by an external loop, is a clean pattern that preserves deterministic enforcement while giving LLM-controlled tasks full tool access.
  • deepagents.task is not a Python module. It's a SubAgentMiddleware method. Tools that need to call sub-agents must return instructions to the LLM rather than calling task() internally.
  • System prompt substring tests are brittle. Adding a BLOCKED protocol to prompts shifted content enough to break 4 assertion tests. Test for semantic intent, not exact substrings.
  • BLOCKED protocol enables structured rollback. EscalationRouter with repeated-path detection + MAX_ESCALATIONS provides defense-in-depth against infinite pipeline loops.
  • Idempotency pattern is essential for re-entrant graphs. Each node checks (output_exists AND NOT needs_rework) before producing a CALL_TASK instruction. This allows the graph to be invoked repeatedly without redundant work.
